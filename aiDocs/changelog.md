**Code complete.** Schema, Drizzle config, migration SQL, seed script, and SSR deploy config in place. `npm run typecheck` and `npm run build` pass.

**Known deviation — Vercel preset:** `@vercel/react-router` (which provides `vercelPreset()`) has a peer dependency on `@react-router/dev@7`. This project uses v8 — the preset cannot be installed. The current `react-router.config.ts` (`ssr: true`, no preset) + `vercel.json` (`framework: null`) is the correct approach for v8 on Vercel.

**Convention — `db:push` vs `db:migrate`:** Use `db:push` during local development (fast, no migration file required). Use `db:generate` + `db:migrate` for any schema change that needs to be tracked or applied against production Neon. The `drizzle/0000_init.sql` file was generated via `db:generate`; apply it to Neon with `db:migrate` (or `db:push` if iterating quickly).

**Deferred — `vercel dev` SSR:** `npm run build` and `npm run typecheck` pass. `vercel dev` SSR confirmation is deferred until a real `DATABASE_URL` is in `.env` and `vercel link` has been run (see Phase 0 notes).

**Deferred — date fields:** No `scheduled_at` or cycle-start date is stored in the schema. Phase 4's "next check-in date" per direct report will be computed in application code (e.g. `CURRENT_WEEK * 7` days from a hardcoded cycle start, or a placeholder string). Adequate for the demo; add a `cycle_started_at` column to `learners` if this needs to be real.
